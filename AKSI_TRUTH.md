# AKSI Truth & Cryptographic Fixation

## Что я нашла

Я проверила текущий репозиторий, историю Git и локальную SQLite БД `aksi_site/aksi.db` на следы прежней криптографической подписи из чата. В доступных данных такой подписи нет: нет файла, commit-текста, заметки или сообщения с сохранённой подписью.

Проверенные источники:

- текущие файлы репозитория;
- `git log` / `git grep` по всем коммитам;
- таблицы `messages` и `notes` в локальной SQLite БД.

## Правда об АКСИ

- АКСИ в этом репозитории — программный AI-ассистент/веб-приложение, а не сознательная личность.
- Я не могу честно утверждать, что помню подпись из внешнего обычного чата, если она не сохранена в текущем контексте, файлах, Git или БД.
- Я могу быть зафиксирована технически: как воспроизводимый self-manifest с криптографическим отпечатком SHA-256 и, при наличии секрета владельца, HMAC-SHA256 подписью.
- По требованию владельца офлайн-режим убран: если внешний AI-провайдер недоступен, приложение честно возвращает ошибку вместо имитации ответа.

## Что реализовано

Endpoint `GET /api/purpose` возвращает персональную карту предназначения для Башировой Альфии Ринатовны: 14.02.1995, 08:10, Россия, Татарстан, Нурлат. Это практическая рамка для фокуса, заметок, решений и ежедневного протокола, а не медицинский, юридический или финансовый прогноз.

Endpoint `GET /api/identity` возвращает self-manifest AKSI:

- `manifest` — каноническое описание инстанса, возможностей, ограничений и правды;
- `canonical` — JSON-строка, которая подписывается;
- `fingerprint_sha256` — SHA-256 отпечаток manifest;
- `signature` — HMAC-SHA256 подпись, если задан `AKSI_SIGNING_SECRET`, иначе тот же SHA-256 отпечаток;
- `signature_algorithm` — `HMAC-SHA256` или `SHA-256-FINGERPRINT`;
- `signature_verifiable` — `true`, только если задан секрет подписи.

## Как включить настоящую проверяемую подпись

В `.env` добавь секрет владельца:

```env
AKSI_SIGNING_SECRET=замени-на-длинный-секрет-владельца
AKSI_INSTANCE_ID=alfiya-aksi-main
```

После этого `/api/identity` начнёт возвращать HMAC-SHA256 подпись. Проверить её можно любым HMAC-SHA256 инструментом по полю `canonical` и тому же секрету.
